# Fleet - Local Dev Environment

## Overview

A fleet of 6 Docker containers simulating a multi-tier application environment, managed via Ansible and AWX over SSH.

## Fleet Composition

| Container | Role | Packages | Ports |
|---|---|---|---|
| `fleet-web-01` | Web server | nginx, curl | SSH :2201, HTTP :8081 |
| `fleet-web-02` | Web server | nginx, curl | SSH :2202, HTTP :8082 |
| `fleet-app-01` | App server | python3-pip, git | SSH :2203 |
| `fleet-app-02` | App server | python3-pip, git | SSH :2204 |
| `fleet-db-01` | Database | postgresql | SSH :2205 |
| `fleet-lb-01` | Load balancer | haproxy | SSH :2206, HTTP :8080 |

All nodes: Ubuntu 22.04, SSH-enabled (root/awxtarget), Python 3.

## Quick Start

```bash
# Build and start the fleet
./fleet/fleet.sh up

# Check status
./fleet/fleet.sh status

# Run Ansible against the fleet
ansible-playbook -i inventory/fleet_hosts.yml playbooks/fleet_configure.yml

# SSH into a node
./fleet/fleet.sh ssh fleet-web-01
```

## Fleet Management CLI

```
Usage: fleet.sh <command> [args]

Fleet Management:
  up                Build and start all fleet nodes
  down              Stop and remove fleet containers
  destroy           Stop, remove containers, images, and volumes
  status            Show fleet container status with IPs
  ssh <node>        SSH into a fleet node
  ips               Show container name -> IP mapping

State Management:
  save [tag]        Snapshot all containers (default tag: timestamp)
  restore <tag>     Restore fleet from a saved snapshot
  list-saves        List available saved states

Ansible Integration:
  inventory         Generate/refresh the Ansible inventory file
```

## State Management

### Save state

Snapshots all running containers and saves them as a compressed tarball.

```bash
# Save with auto-generated timestamp tag
./fleet/fleet.sh save

# Save with a named tag
./fleet/fleet.sh save configured
./fleet/fleet.sh save after-deploy
```

### Restore state

Destroys the current fleet and restores from a saved snapshot, preserving all filesystem changes (installed packages, config files, deployed apps).

```bash
# Restore from a named tag
./fleet/fleet.sh restore configured

# Restore the latest save
./fleet/fleet.sh restore latest

# List available saves
./fleet/fleet.sh list-saves
```

### How it works

1. **Save**: `docker commit` each container into a snapshot image, then `docker save` all into a single `.tar.gz`
2. **Restore**: `docker load` the tarball, generate a `docker-compose.override.yml` that swaps build images for snapshot images, then `docker compose up`

Snapshots are stored in `fleet/saved-states/` and excluded from git.

## Network Architecture

The fleet containers are connected to two Docker networks:

- **`kind`**: Shared with the AWX kind cluster, allowing AWX to reach fleet nodes via internal IPs
- **`fleet`**: Bridge network for host-side access via mapped ports

```
Host Machine
├── localhost:2201-2206  →  SSH to fleet nodes
├── localhost:8080-8082  →  HTTP to web/lb nodes
└── Docker Networks
    ├── kind (external)
    │   ├── awx-cluster-control-plane
    │   ├── awx-target-node
    │   └── fleet-* containers
    └── fleet (bridge)
        └── fleet-* containers
```

## Ansible Inventory

The inventory file `inventory/fleet_hosts.yml` is auto-generated by `fleet.sh` and uses localhost with mapped SSH ports:

```yaml
fleet:
  children:
    fleet_web:
      hosts:
        fleet-web-01:
          ansible_host: 127.0.0.1
          ansible_port: 2201
        fleet-web-02:
          ansible_host: 127.0.0.1
          ansible_port: 2202
    fleet_app: ...
    fleet_db: ...
    fleet_lb: ...
```

Regenerate after any fleet restart: `./fleet/fleet.sh inventory`

## Playbooks

| Playbook | Target | Description |
|---|---|---|
| `fleet_configure.yml` | `fleet` | Master playbook: baseline + imports role-specific plays |
| `fleet_web.yml` | `fleet_web` | Deploys nginx index page, starts nginx |
| `fleet_app.yml` | `fleet_app` | Creates `/opt/app/`, deploys sample Python app |
| `fleet_db.yml` | `fleet_db` | Initializes PostgreSQL cluster |

## AWX Integration

The fleet nodes are registered in AWX under the "Docker Nodes" inventory with groups matching the Ansible inventory structure (`fleet` > `fleet_web`, `fleet_app`, `fleet_db`, `fleet_lb`).

AWX Job Template: **"Configure Fleet - All"** (ID: 11)

AWX connects to fleet nodes via their `kind` network internal IPs.
